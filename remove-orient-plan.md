# Plan: Remove `orient` Command and `lexi-orient` Skill

## Context

The `lexi orient` command aggregates topology, library stats, and IWH signals into a single output for agent session start. Its responsibilities are already covered by reading `TOPOLOGY.md` directly and `lexi iwh list` for signals. The command is being fully removed -- not deprecated -- to simplify the CLI surface and agent workflow.

**Key finding:** The `core_rules.md` template references `lexi orient` at line 5 and this propagates into `CLAUDE.md` (line 62) and `AGENTS.md` (line 46) via the managed `<!-- lexibrary:start -->` section. All three must be updated.

**Replacement workflow for agent templates:** The canonical replacement block is:

```markdown
## Session Start

Read `.lexibrary/TOPOLOGY.md` for project layout, then run `lexi iwh list`
to check for pending signals. Do not begin coding without this context.

If signals exist in a directory you are working in, run `lexi iwh read <dir>` to
consume the signal and understand what the previous session left behind.

Do not consume IWH signals for directories you are not actively working in —
consuming deletes the signal permanently. Sub-agents must not consume IWH signals.
Do not create an IWH signal if all work is complete.
```

Use this exact block when replacing orient references in `core_rules.md`, agent templates, and dogfood copies.

**Commit strategy:** All changes across all phases should be committed as a single atomic commit.

## Risks

1. **Test volume** (~300+ lines across 11 files) -- mitigated by merging tests into Phase 2 and running per-file.
2. **Agent template coherence** -- 3 templates + 3 dogfood copies must have identical replacement workflow. Mitigated by the canonical replacement block defined at the top of this plan.
3. **Init rules generation** -- Phases 2.4-2.7 modify rule generators that ship to OTHER projects. Integration tests must pass.
4. **Generated files** -- `CLAUDE.md`, `AGENTS.md`, and `.lexibrary/TOPOLOGY.md` are regenerated by `lexictl setup --update`. User runs this post-commit.
5. **Stale orient skill cleanup** is out of scope -- older projects with `.claude/skills/lexi-orient/` will need to remove it manually. The old `.claude/commands/lexi-orient.md` is covered by `_STALE_COMMAND_FILES`. A `_STALE_SKILL_DIRS` mechanism for skill directory cleanup is a future improvement.
6. **Existing user configs** -- Projects with `orientation_tokens` in their `config.yaml` will not break (`ConfigDict(extra="ignore")` silently drops unknown fields). No migration action needed.

## Follow-up: Improve Config Validation Error Messages

**Out of scope for this plan, but recommended.**

`load_config()` in `src/lexibrary/config/loader.py` currently lets `pydantic.ValidationError` propagate raw (line 135). When a config file has invalid values, users see a Pydantic traceback with no indication of which file or YAML line is at fault.

**Recommended improvement:** Catch `ValidationError` in `load_config`, enrich with the source file path (global vs project) and the offending field path/value, and re-raise with a user-friendly message. This would help with any future config field changes, not just orient removal.
