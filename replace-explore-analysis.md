## What This Change Does

Replaces the built-in Claude Code Explore subagent with a Lexibrary-aware version using two complementary strategies:

- **Strategy 2 (SubagentStart Hook)**: `.claude/hooks/lexi-explore-context.sh` — automatically injects `lexi context-dump` output into Explore/Plan subagents as `additionalContext`
- **Strategy 3 (Custom Agent Override)**: `.claude/agents/explore.md` — overrides the built-in Explore agent with a Lexibrary-aware prompt that has Bash tool access for running `lexi` commands

The design chose **both strategies combined**, reasoning that neither alone is sufficient: the hook can't give the agent tools, and the custom agent alone doesn't get pre-loaded context.

### Finding 3: Haiku Completely Ignores Soft Instructions

Despite the custom agent prompt saying "Prefer Lexibrary commands over raw Glob/Grep" and CLAUDE.md saying "Run `lexi lookup <file>` before editing", the Haiku model:
- Used `find`, `ls -la`, `cat` via Bash for filesystem exploration
- Used `Read` for every file access
- Used `Glob` and `Grep` for searching
- **Never ran a single `lexi` command**
- User had to manually reject a `cat` call and type "use lexi instead" to get any lexi usage

This is the core problem: Haiku does not reliably follow "prefer X over Y" style instructions, especially when familiar tools (Read, Grep, Glob) are available.

## Fix Applied: Assertive Agent Prompt

Updated `_EXPLORE_AGENT_CONTENT` in `src/lexibrary/init/rules/claude.py` (lines 185-237) and the generated file at `/Users/shanngray/AI_Projects/TEMP/AdAM/.claude/agents/explore.md`.

Key changes:
1. **Removed Glob and Grep from tools list** — agent now only has `Read` and `Bash`, physically preventing it from using Glob/Grep
2. **Changed "Prefer" to "MUST"** — "You MUST use `lexi` CLI commands as your PRIMARY exploration tools. Do NOT use Glob or Grep."
3. **Added "CRITICAL" section** with explicit required workflow (start with `lexi search`, use `lexi lookup` before reading files)
4. **Gated Read tool** — "Use the Read tool ONLY after you have identified the right file via `lexi search` or `lexi lookup`"
5. **Moved fallback behind an explicit check** — "If `.lexibrary/` does not exist... then and ONLY then fall back"

## Key Insights

### The "Prefer" Problem
Haiku (and likely other smaller models) treat "prefer X" as a weak suggestion. When familiar tools are available, the model defaults to what it knows. For custom agent prompts targeting Haiku, instructions must be:
- Imperative (MUST, NEVER, CRITICAL)
- Structurally enforced (remove tools you don't want used)
- Workflow-ordered (numbered steps, not guidelines)

### Custom Agent Override Works, But Is Not Enough Alone
The override mechanism (`.claude/agents/explore.md` with `name: Explore`) correctly replaces the built-in agent. But giving the agent both lexi commands AND raw tools means it will always choose the familiar raw tools. The fix is to remove raw tools and force the lexi path.

### SubagentStart Hooks Don't Work in v2.1.63
Confirmed across two test sessions: PreToolUse/PostToolUse hooks fire correctly inside subagents, but SubagentStart never fires. This is either not implemented or broken. Strategy 2 (hook-based context injection) should be removed or deferred until a Claude Code version confirms SubagentStart support.

### Strategy 3 Alone Is Sufficient
The custom agent override with assertive prompting achieves all goals without the SubagentStart hook. The agent gets its lexi awareness from the prompt itself + Bash tool access, making pre-injected context unnecessary.
