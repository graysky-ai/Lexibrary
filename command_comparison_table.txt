COMPREHENSIVE COMMAND COMPARISON
==========================================================

COMMAND SIGNATURES
==========================================================

lexi search [query] [--tag TAG] [--scope SCOPE]
  - Unified search across all artifact types
  - At least one of query/tag/scope required

lexi concepts [topic] [--tag TAG...] [--status STATUS] [--all]
  - Concept vocabulary discovery
  - Repeatable --tag (AND logic)

lexi conventions [query|path] [--tag TAG...] [--status STATUS] [--scope SCOPE] [--all]
  - Convention project rules
  - Dual-mode positional arg (path detection via "/" or ".")
  - Repeatable --tag (AND logic)

lexi stack search [query] [--tag TAG] [--scope SCOPE] [--status STATUS] 
                  [--concept CONCEPT] [--resolution-type TYPE] [--include-stale]
  - Stack issue discovery
  - Specialized filters: concept, resolution-type

lexi stack list [--status STATUS] [--tag TAG] [--include-stale]
  - Stack issue browsing (no query)
  - Minimal filters

OUTPUT FORMATS
==========================================================

lexi search:
  - Four Rich tables (one per artifact type), groups with 0 results omitted
  - Concepts:       Name, Status, Tags, Summary
  - Conventions:    Title, Scope, Status, Rule, Tags
  - Design Files:   Source, Description, Tags
  - Stack Posts:    ID, Status, Votes, Title, Tags

lexi concepts:
  - Single Rich table: Name, Status, Tags, Summary + count footer
  - Deprecated excluded by default (--all to include)

lexi conventions:
  - Single Rich table: Title, Scope, Status, Tags, Rule + count footer
  - Deprecated excluded by default (--all to include)

lexi stack search / list:
  - Single Rich table: ID, Status, Votes, Title, Tags + help tip if no results
  - Stale excluded by default (--include-stale to show)


LINK GRAPH ACCELERATION
==========================================================

lexi search (when link graph available):
  - Tag searches: O(1) index lookup via tags table
  - Free-text: FTS5 full-text search with relevance ranking
  - Fallback: file-scanning when link graph missing

lexi concepts / conventions / stack:
  - No documented link graph acceleration
  - Always use file-scanning via respective Index classes


UNIQUENESS & CANNOT-REPLACE FEATURES
==========================================================

  (--concept, --resolution-type)                       No equivalent in other types

  filtering                          (& concepts)       concepts, different semantics

CONSOLIDATION ANALYSIS
==========================================================

CAN stack list MERGE TO stack search?
  Status: MOSTLY YES (minor consolidation candidate)
  Current: "stack list --status open" ≈ "stack search --status open"
  Cons: Loses semantic clarity (list vs search is meaningful to users)
  Verdict: Could merge but not recommended

CAN lexi search REPLACE ALL OTHERS?
  Status: NO (major loss of specialized functionality)
  Would lose:
    1. Path-scoped convention inheritance
    2. AND-tag filtering per artifact type
    3. Stack concept/resolution-type filters
    4. Semantic clarity for specialists
    5. Simple fast list operations
  Verdict: Do not consolidate

CAN lexi conventions REPLACE lexi search?
  Status: NO (only covers conventions, missing other 3 artifact types)
  Verdict: Not applicable
