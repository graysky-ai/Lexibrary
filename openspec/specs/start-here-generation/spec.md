# start-here-generation Specification

> **REMOVED** -- The LLM-based START_HERE generation pipeline was removed by
> change [`start-here-dismantling`](../../changes/start-here-dismantling/design.md).
> The `generate_start_here()` function, `StartHereOutput` BAML type, and
> `start_here_tokens` config field have all been deleted. The replacement is
> `generate_topology()` in `src/lexibrary/archivist/topology.py`, which
> procedurally builds `.lexibrary/TOPOLOGY.md` from `.aindex` data without
> any LLM calls. This spec is retained for historical reference only.

## Requirements
### Requirement: generate_start_here function
The system SHALL provide `async generate_start_here(project_root, config, archivist) -> Path` in `src/lexibrary/archivist/start_here.py` that generates or updates `.lexibrary/START_HERE.md` from project topology and `.aindex` summaries.

The pipeline SHALL:
1. Build directory tree string (exclude `.lexibrary/`, ignored dirs)
2. Collect billboard summaries from all `.aindex` files in the mirror tree
3. Read existing `START_HERE.md` if it exists (for continuity)
4. Call `archivist.generate_start_here()`
5. Assemble final markdown from StartHereOutput sections — excluding `convention_index`
6. Append a conventions pointer: "Run `lexi lookup <file>` to see applicable conventions, or `lexi conventions` to list all project conventions."
7. Validate against token budget (`start_here_tokens`)
8. Write to `.lexibrary/START_HERE.md`

The assembly function SHALL NOT include a `convention_index` section from the LLM output. Conventions are managed as a separate first-class artifact.

#### Scenario: Generate START_HERE for project
- **WHEN** `generate_start_here()` is called
- **THEN** `.lexibrary/START_HERE.md` SHALL be written with topology, ontology, navigation, and protocol sections, plus a conventions pointer

#### Scenario: START_HERE does not contain hallucinated conventions
- **WHEN** `generate_start_here()` completes
- **THEN** the output SHALL NOT contain a `## Convention Index` section generated by the LLM

#### Scenario: START_HERE contains conventions pointer
- **WHEN** `generate_start_here()` completes
- **THEN** the output SHALL contain text directing users to `lexi lookup` and `lexi conventions` for convention information

#### Scenario: Collects .aindex summaries
- **WHEN** `generate_start_here()` builds input for the LLM
- **THEN** it SHALL read billboard text from each `.aindex` in the mirror tree

#### Scenario: Excludes .lexibrary directory from tree
- **WHEN** `generate_start_here()` builds the directory tree
- **THEN** the `.lexibrary/` directory SHALL NOT appear in the tree string

#### Scenario: Token budget warning
- **WHEN** the generated START_HERE.md exceeds `start_here_tokens` budget
- **THEN** a warning SHALL be logged

### Requirement: START_HERE.md regeneration triggers
`START_HERE.md` SHALL be regenerated when:
- `lexi update` runs without a specific path (full project update)
- The set of top-level directories has changed since last generation

`START_HERE.md` SHALL NOT be regenerated on every single file update.

#### Scenario: Full project update regenerates START_HERE
- **WHEN** `lexi update` runs with no path argument
- **THEN** `START_HERE.md` SHALL be regenerated after all file updates

#### Scenario: Single file update skips START_HERE
- **WHEN** `lexi update src/foo.py` runs (single file)
- **THEN** `START_HERE.md` SHALL NOT be regenerated

### Requirement: HANDOFF.md not generated by Archivist
The Archivist SHALL NOT generate or modify `HANDOFF.md`. It is written exclusively by agents during their sessions. `lexi init` creates a placeholder only.

#### Scenario: Archivist leaves HANDOFF alone
- **WHEN** `update_project()` runs
- **THEN** `HANDOFF.md` SHALL NOT be created or modified by the Archivist pipeline

### Requirement: BAML prompt excludes convention_index
The `archivist_start_here.baml` prompt SHALL NOT ask the LLM to generate a `convention_index` field. The `StartHereOutput` type in `types.baml` SHALL NOT include a `convention_index` field.

#### Scenario: StartHereOutput has no convention_index
- **WHEN** the `StartHereOutput` BAML type is inspected
- **THEN** it SHALL NOT contain a `convention_index` field

#### Scenario: BAML prompt does not mention convention extraction
- **WHEN** the `archivist_start_here.baml` prompt is inspected
- **THEN** it SHALL NOT contain instructions to generate a `convention_index`
