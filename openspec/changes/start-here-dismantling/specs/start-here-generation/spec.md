## REMOVED Requirements

### Requirement: generate_start_here function
**Reason**: START_HERE.md is replaced by procedural TOPOLOGY.md. The LLM-generated artifact is removed entirely — topology comes from `.aindex` data, ontology/navigation-protocol/convention-index are redundant or have no backing data source.
**Migration**: Pipeline step 5 calls `generate_topology()` from `topology.py` instead of `generate_start_here()` from `start_here.py`. Agents reference `.lexibrary/TOPOLOGY.md` instead of `.lexibrary/START_HERE.md`.

### Requirement: START_HERE.md regeneration triggers
**Reason**: TOPOLOGY.md regeneration is handled by the new `procedural-topology` capability. The triggers remain the same (full project update), but the implementation moves to `topology.py`.
**Migration**: See `procedural-topology` spec and `archivist-pipeline` delta spec.

### Requirement: HANDOFF.md not generated by Archivist
**Reason**: This requirement is unaffected by the change — HANDOFF.md was never part of START_HERE generation. It remains valid but does not need to be in the start-here-generation spec since that spec is being removed.
**Migration**: No migration needed. The requirement still holds and is documented in the archivist-pipeline spec.
